封闭、杂乱的微信公众号文章如何高效阅读

序言

互联网的缔造者们花费大量的心血制定各项标准与规范以促进网络技术的开放和协同发展。
而有些互联网公司非但没有因站在巨人肩膀上保持该有的低调和谦虚，反倒肆无忌惮的人为设置各项障碍以维护自身脆弱的护城河，达到占据用户资源、消费用户时间、侵占用户隐私的不良勾当，还美其名曰大数据营销，实在是非常营销。

关于微信公众号的吐槽

微信公众号就是上述非常营销作品中最有影响力的一个，文章的导入导出、搜索、跨平台阅读、标注与笔记摘抄、批量操作等最基本的知识管理操作体验简直是十足的噩梦。
只因它与即时聊天工具的绑定，成就了其国内最重要的信息源之一。

我所关注的公众号统计

竭尽所能的控制下，我还是有 110 多个公众号需要关注，统计一下发现，这 110 多个微信公众号每周大概发布 160 多篇文章，每月将近 800 篇。
如若不解决批量操作、高效摘抄等知识管理需求的障碍，所剩的选择酒只有两个。要么断舍离，佛系的忽略，任由信息的流失，要么就只能深陷在狭小的手机屏幕上，花费大量的时间各种打开、点击、打开、点击、收藏……等无止境的手指操作。

解决方案

面对上述情况，经历了各种失败后，有了以下思路，暂且突破这一窘境，初步实现了微信公众号文章高效自主的阅读。

申请一个微信公众号，没错，还要申请一个微信公众号。

打开微信公众号管理页面-新增图文素材，在顶层操作栏，可以看到一个"超链接"按钮。

按 F12 打开 Chrome 开发者工具，点击 "超链接" 按钮，在 Network 标签下即可看到目前唯一可找到的获取微信公众号文章的开放 API 和请求参数（经测试发现，只需 key 和 Cookie 两个参数即可）。

基于自身熟悉的编程语言，结合该 API 与请求参数，即可开发批量爬取程序。

将爬取到的文章链接批量 post 至 Feedly，实现微信公众号与 RSS 阅读工具的统一集成，从而串起后续自动化、批量化的知识管理操作。

总结

即便如此也要控制好爬取的频率。我有将近 90 个公众号需要爬取，每周爬取一次，中间会有一次超出爬取上限，停留大概半个小时即可进行二次爬取，基本上都要分两次才能爬完。
哈哈，是有点复杂，不过熟悉之后，可节约大量碎片化时间，用以解放身心，专注其他更有意义的事情了。
